--------------------------------------------------------------------------------
② 트렌드 없이도 강한 주제 설계법 (3번 훑기)

이거 중요하다.
트렌드 없이도 잘 되는 채널들의 공통점은 이거야 👇

❌ “지금 핫한 것”
✅ “항상 반복되는 불안과 욕망”

🔥 강한 주제의 4가지 타입 (알고리즘 친화)
① 구조적 불안형 (금융 최강)

“이러다 ○○ 된다”

“대부분이 놓치고 있는 리스크”

📌 예:

“지금 미국 금리, 진짜 문제는 이거다”

② 오해 교정형

사람들이 잘못 믿고 있는 것

반전이 있음

📌 예:

“비트코인 반등? 진짜 이유는 따로 있다”

③ 시기 질문형

“지금이 맞나?”

타이밍 불안

📌 예:

“지금 집 사도 될까? 데이터로 보면…”

④ 결과 압축형

복잡한 걸 한 문장으로 요약

📌 예:

“결론부터 말하면, 지금은 ○○다”

👉 이건 트렌드 필요 없음
👉 인간 심리 기반이라 항상 먹힘

🧠 그래서 시스템에서는 이렇게 처리

TrendScout 없을 때:

User Topic
  ↓
topic_research
  ↓
viewer_intent_analysis
  → 불안 / 욕망 / 질문 중 하나 강제 선택
  ↓
ctr_package_skill


👉 주제를 “심리 프레임”으로 재가공

✅ CTO가 잡은 절대 우선순위 (변경 없음)
1️⃣ PROJECT_SPEC 고정

“이 시스템은 무엇을 하고, 무엇을 절대 하지 않는가”

프롬프트 흔들림

나중에 전부 갈아엎음

👉 가장 먼저

2️⃣ 단일 Agent + Skill Pipeline 확정

Agent는 판단만, Skill은 일만 한다

판단과 실행을 분리해야

확장 가능

디버깅 가능

자동 평가 가능

👉 두 번째

3️⃣ CTR 패키지 스킬 선구현

조회수 안 나오는 시스템은 아무 의미 없음

훅

제목

썸네일 문구

이 3개가:

잘 나오면 → 시스템은 살음

안 나오면 → 전부 무의미

👉 세 번째

4️⃣ Eval → 실패 시 재작성

“AI가 스스로 쪽팔린 결과를 버릴 수 있어야 한다”

이게 들어가면

체감 퀄리티 급상승

인간 개입 최소화

👉 네 번째
